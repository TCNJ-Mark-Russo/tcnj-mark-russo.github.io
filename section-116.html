    Section 13.3 Runtime Errors

    The program compiles, but something goes wrong while it is running. Java responds to the problem by throwing an exception. Like nearly everything else in Java, an exception is an object created by instantiating a suitable exception class that identifies the problem.
    If there is no intervention, Java will eventually print the current state of the call stack when the exception object is thrown.
