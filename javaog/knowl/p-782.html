But modifying a class with abstract is not enough. If a class is abstract then it must also contain at least one method that is modified with abstract as well. A method with the additional abstract modifier is required to have no implementation. Instead, an abstract method is composed of a method signature only, and this signals that all derived classes must provide a concrete implementation of the method. Once again, the Java compiler enforces this design constraint. Classes that extend an abstract base class must (eventually) provide a concrete implementation of all abstract methods.
