Consider what might happen if we decided to leave the initialization of the LED object’s instance variables by setting the object state explicitly in the constructor. Doing this requires us to make sure that the constructor and the turnOff() method remained in sync. If we decided to change the fill color of the LED when off, and we modified the code in turnOff() but forgot to make the comparable change to the constructor, we would end up with an inconsistency in our program. The ramifications of making this mistake in this example are minor. But one can imagine how such an oversight can lead to more serious problems. This important software development principle has been dubbed Don’t repeat yourself or DRY.
