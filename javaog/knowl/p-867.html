The format that we’ll work with here is often referred to as plain text files. The way to interpret the sequence of bytes in a plain text file is as encoding the sequence of characters in the file. The most common standard for translating between the numerical byte value and its character equivalent is the ASCII (See Appendix A). To interpret the contents of a plain text file, read the byte sequence and substitute each value with an equivalent character from the ASCII lookup table. Writing a plain text file is the inverse process. Convert your file contents to a String, and then write the ASCII bytes to a file that correspond to the character sequence of that String.
