You may provide multiple catch-blocks, each with different exception class variable declarations. The first declaration that matches the thrown exception is the block that gets executed, if any. If a different exception object is thrown by the example above, the catch-block would not match and Java would resort to dumping the stack. To avoid this kind of problem, it can be useful to add a final catch-block that catches an Exception base class. This way we know that all exceptions will be caught, no matter which one is thrown, because all exception objects can be assigned to a variable with base class type Exception.
