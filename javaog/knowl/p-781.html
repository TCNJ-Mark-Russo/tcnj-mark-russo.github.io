If Shape is abstract, then it must have at least one abstract method declared. Of course it does. The one abstract method of Shape is draw(…), which should make some sense. The one unique feature of all Shape’s derived classes it their graphical representation. The draw(…) method is different for each derived class of Shape, and must be implemented. Modifying draw(…) with abstract enlists the compile to enforce the requirement that all derived classes must implement draw(…).
