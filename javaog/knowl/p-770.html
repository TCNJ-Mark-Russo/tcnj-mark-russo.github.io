Protected visibility helps us to eliminate the need to duplicate members in a derived class that are defined already in a base class. In Listing 9.4.3 both Base and Derived declared the private instance variable name. This is unnecessary duplication. In Listing 9.4.7 we change the visibility of the derived class name instance variable to protected, which makes the variable visible to the derived class as well. With this enhanced level of visibility, we can not remove the Derived instance variable name altogether, because Derived can use the one declared in Base. This change in design is more succinct and reflective of the implications of inheritance. If we compile and run this updated program, the outcome is unchanged.
