ListingÂ 8.3.2 shows the code changes to our class definition. The added turnOn() and turnOff() methods do what we expect, setting and configuring the state of the object by modifying its private instance variables. The isOn() method returns the boolean value of the on instance variable. Because we have abstracted the concept of the LED being off and on and exposed that as these new methods, it is wise to remove the initial explicit instance variable configuration statements from the constructor that starts the LED in the off state and replace them by invoking the turnOff() method directly. This way, should turnOff() be modified later to do more than its current definition, we can feel confident that making the update in one place will apply whenever the LED is turned off.
