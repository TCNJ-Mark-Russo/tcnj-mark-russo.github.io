The last broad category of errors occurs after a program compiles, but something goes wrong while it is running. Java responds to such problem by throwing an exception. Like nearly everything else in Java, a thrown exception is a type of object created by instantiating a suitable Exception class. A thrown exception object may come from code in the Core Library, or you may instantiate and throw exceptions on your own. The particular Exception class instantiated identifies the detected problem.
