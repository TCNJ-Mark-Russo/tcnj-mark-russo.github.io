The Importance of Testing.

The compiler tells us if we have syntax errors and the execution engine throws exceptions when something goes wrong while the program is running (runtime errors). How can we check when the program runs without error, but produces incorrect results (logic errors)? The best way to check for logic errors is through testing.
A good test plan executes all paths through your source code and presents a wide range of possible inputs. It checks results produced to make sure all they are correct.
