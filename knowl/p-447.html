Taking the concept of an abstract class to the extreme, where all defined methods are signatures only, no implementation, we end up with a new Java concept called an interface. Unlike base classes, which are extended by another class, interfaces are implemented by a class. Because an interface is composed entirely of method signatures and no implementations, implementing a Java interface can be thought of as a contract that must be fulfilled by the implementing class. To fulfill the contract, the implementing class must implement all the signatures in the interface.
